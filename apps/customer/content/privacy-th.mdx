## การเก็บ IP Address ใน Admin Audit Log

### วัตถุประสงค์

เราเก็บรวบรวม IP address ของผู้ดูแลระบบ (Admin) เมื่อมีการดำเนินการผ่านระบบจัดการหลังบ้าน เพื่อ:
- ตรวจสอบความปลอดภัยของระบบ (Security Monitoring)
- ป้องกันการทุจริตในการดำเนินการของผู้ดูแลระบบ (Anti-fraud)
- สอบสวนเหตุการณ์ด้านความปลอดภัย (Incident Response)

### ฐานทางกฎหมาย

การเก็บรวบรวมข้อมูลนี้อยู่บนฐานของ **ประโยชน์อันชอบธรรม (Legitimate Interest)**:
- **พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA)** มาตรา 24(5) — ฐานประโยชน์อันชอบธรรม
- **GDPR** Article 6(1)(f) — Legitimate Interests

### ระยะเวลาการเก็บรักษา (Retention)

| ระยะเวลา | การจัดการ IP Address |
|-----------|---------------------|
| 0–30 วัน | เก็บ IP Address แบบเต็ม (Raw) เพื่อใช้สอบสวนเหตุการณ์ |
| 31–90 วัน | ปิดบังบางส่วน (Masked) — IPv4 เหลือ /24, IPv6 เหลือ /48 |
| มากกว่า 90 วัน | ลบ IP Address ออก (ตั้งค่าเป็น NULL) — บันทึก audit log ยังคงเก็บไว้ |

### สิทธิของเจ้าของข้อมูล

คุณมีสิทธิ:
- **สิทธิในการเข้าถึง (Right of Access)** — ขอดูข้อมูล IP address ที่เราเก็บรวบรวมเกี่ยวกับคุณ
- **สิทธิในการคัดค้าน (Right to Object)** — คัดค้านการเก็บรวบรวม IP address

หากต้องการใช้สิทธิ กรุณาติดต่อเราผ่าน [แบบฟอร์มคำขอข้อมูลส่วนบุคคล](/data-request)
